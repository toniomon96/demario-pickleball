## Section 1: Bug & Functionality

### BUG-01 — Booking submission fails 100% of the time
- **Severity:** Critical / Launch Blocker
- **File:** [src/app/api/bookings/route.ts](src/app/api/bookings/route.ts#L41-L56)
- **Current state:** The POST endpoint uses an anonymous Supabase client (`anonClient()`) to insert rows, but no RLS INSERT policy exists for anonymous users on the `bookings` table. Every submission returns HTTP 500.
- **Error shown to user:** `"new row violates row-level security policy for table 'bookings'"` — a raw database error message exposed directly in the UI.
- **Recommended fix:**
  - Option A (quickest): Add an RLS policy in Supabase: `CREATE POLICY "allow_anon_insert" ON bookings FOR INSERT TO anon WITH CHECK (true);`
  - Option B (more secure): Switch the POST route to use the service role key, which bypasses RLS, and validate inputs server-side (already done).
  - Additionally, replace the raw `error.message` in the response with a generic user message: `"Booking failed. Please try again or call (469) 371-9220."`
- **Business impact:** Zero revenue. Every single interested student sees an error.

### BUG-02 — Hamburger menu button does nothing
- **Severity:** High
- **File:** [src/components/Nav.tsx](src/components/Nav.tsx#L10)
- **Current state:** The `<button className="menu-btn">` has no `onClick` handler and no associated menu component. Clicking it has zero effect.
- **Recommended fix:** Either implement a slide-out nav drawer with links to page sections (Lessons, About, Pricing, Book), or remove the button entirely until it's built.
- **Business impact:** Broken UI erodes trust. Mobile users expect the hamburger to do something.

### BUG-03 — "Watch" button scrolls to testimonials, not a video
- **Severity:** Medium
- **File:** [src/components/Hero.tsx](src/components/Hero.tsx#L43)
- **Current state:** `<a href="#testimonials">Watch</a>` with a play-button icon. Scrolls to text testimonials with no video content.
- **Recommended fix:** Either add a real video (YouTube/Vimeo embed or short reel) and link there, or rename the button to "See Results" or "Read Reviews" to match what it actually does.
- **Business impact:** Misleads users; undermines coach's credibility.

### BUG-04 — No back-navigation in booking modal
- **Severity:** Low
- **File:** [src/components/BookingModal.tsx](src/components/BookingModal.tsx#L193)
- **Current state:** Once you advance from the form to the date/time picker (step 2), there is no "Back" button to change your name, email, or lesson type. Only "Close" (cancels entirely).
- **Recommended fix:** Add a back chevron in the picker step header to return to `step = "form"`.
- **Business impact:** Minor friction; users who pick the wrong lesson type must close and restart.

### BUG-05 — Default selected time is always "5:30 PM" regardless of context
- **Severity:** Low
- **File:** [src/components/BookingModal.tsx](src/components/BookingModal.tsx#L42)
- **Current state:** `useState("5:30 PM")` hard-sets the default time selection. On Saturday morning, 5:30 PM is pre-selected.
- **Recommended fix:** Default to the first available (non-booked) time slot.

### BUG-06 — Favicon missing, console error on every page load
- **Severity:** Low
- **File:** `public/` (missing `favicon.ico`)
- **Current state:** Console logs `Failed to load resource: 404 Not Found @ /favicon.ico` on every page visit.
- **Recommended fix:** Add a favicon. Minimum: a 32×32 PNG of the "D" brandmark. Ideal: full favicon set via realfavicongenerator.net.

## Section 3: Business & Bookings Logic

### BIZ-01 — No payment collected at booking — zero financial commitment
- **Severity:** Critical
- **Current state:** Students can request any booking time with no credit card, no deposit, no payment. This produces maximum no-show rate (industry standard for free-to-book coaching: 30-50% no-show).
- **Recommended fix:** Integrate Stripe. Two options:
  - **Deposit model:** Collect a $25–$50 deposit at booking, apply to session. Best for retention.
  - **Pay-in-full model:** Full $90/$125/$55 collected at booking via Stripe Checkout or Payment Element.
  - At minimum, add "card on file" via Stripe's SetupIntent to charge no-shows.
- **Business impact:** At current prices, a 40% no-show rate costs DeMario ~$36–$50/missed slot.

### BIZ-02 — Availability is hardcoded, coach cannot manage schedule
- **Severity:** High
- **File:** [src/lib/data.ts](src/lib/data.ts#L94)
- **Current state:** `BOOKED = new Set(["9:00 AM", "3:00 PM"])` — these same two slots are blocked on every day forever. No admin interface to open/close slots, mark holidays, or set custom hours.
- **Recommended fix:** Add an `availability` table in Supabase. Admin dashboard gets a weekly calendar grid where DeMario can toggle each slot open/closed per day. The booking modal reads live availability from a `/api/availability` endpoint.

### BIZ-03 — No cancellation or rescheduling policy enforced
- **Severity:** High
- **Current state:** No cancellation policy is shown anywhere. No rescheduling mechanism exists. Students must email/call to cancel. No late-cancellation fee mechanism.
- **Recommended fix:**
  - State policy in the booking modal: "Cancellations within 24 hours forfeit the deposit."
  - Enforce via Stripe: capture card at booking, charge no-show fee automatically.
  - Add a "Reschedule" flow in a future version.

### BIZ-04 — No lesson packages or upsell path
- **Severity:** Medium
- **Current state:** Only single-session booking. No "5-pack" or "10-pack" discounts. No recurring lesson scheduling. No clinic registration.
- **Recommended fix:** Add lesson packages to the Lessons section and booking modal. A 5-pack at 10% off ($405 for Foundations, $562 for Strategy Lab) dramatically improves LTV and reduces churn.

### BIZ-05 — No coach-facing booking confirmation
- **Severity:** High
- **Current state:** When a booking is created, DeMario only knows about it if he manually checks `/admin`. No push notification, no email, no SMS.
- **Recommended fix:** Send an email to `demariomontez10@gmail.com` on every new booking via Resend or Nodemailer. Include: student name, email, phone, lesson type, date, time.

### BIZ-06 — Booking only shows 8 days of availability
- **Severity:** Medium
- Already noted in UX-02. Cross-referencing here: business impact is inability to pre-sell lesson packs or schedule recurring sessions.

### BIZ-07 — No reviews linked to a real platform
- **Severity:** Medium
- **Current state:** Hero claims "5.0★ · 79 reviews" but there is no link to Google, TeachMe.To, or any review source. Students cannot verify.
- **Recommended fix:** Link the star rating to the actual TeachMe.To coach profile or Google Business Profile. Adds credibility instantly.

## Section 5: Finance & Accounting

### FIN-01 — No revenue collected through the site
- **Severity:** Critical
- **Current state:** The site is a booking request system, not a payment system. All revenue collection happens offline (presumably cash/Venmo after the lesson). This is a structural problem:
  - No deposit = no commitment = high no-show rate
  - No card on file = no ability to charge late cancellations
  - No payment record = no clean accounting
- **Recommended fix:** Stripe integration. Estimated implementation: 4–8 hours.
  - `npm install stripe @stripe/stripe-js @stripe/react-stripe-js`
  - Add `STRIPE_SECRET_KEY` and `STRIPE_PUBLISHABLE_KEY` to `.env.local`
  - Create a `/api/checkout` route that generates a Stripe Checkout session
  - Redirect booking confirmation to Stripe Checkout, then back to a `/booking-confirmed?session_id=` page
  - Store `stripe_payment_intent_id` on the booking record

### FIN-02 — No bookkeeping integration or transaction record
- **Severity:** Medium
- **Current state:** Bookings are stored with lesson type and price as display strings (`"$90.00"`), not as integers. No `amount_paid`, `payment_status`, or `payment_intent_id` fields on the bookings table.
- **Recommended fix:** Add `amount_cents INTEGER`, `payment_status TEXT`, `stripe_payment_intent_id TEXT` to the bookings schema before launch. This enables clean QuickBooks / Wave import.

### FIN-03 — No invoice or receipt sent to students
- **Severity:** Medium
- **Current state:** No receipt is generated or emailed after a booking/payment.
- **Recommended fix:** Stripe automatically generates receipts when configured. Enable `receipt_email` on the PaymentIntent and set it to the student's email.

### FIN-04 — No cancellation fee enforcement mechanism
- **Severity:** High
- **Current state:** No way to charge a no-show or late cancellation fee — no card on file.
- **Recommended fix:** Even if not charging upfront, use Stripe's SetupIntent to save a card at booking. Policy: "Card is saved. 24-hour cancellation required or $30 cancellation fee applies."

### FIN-05 — No lesson package / pre-paid revenue model
- **Severity:** Medium
- **Current state:** Single-session only. Pre-paid packs (5-pack, 10-pack) are a coaching industry standard that provide:
  - Cash flow (paid upfront, service delivered later)
  - Retention (students who bought a 5-pack come back)
  - Accounting simplification
- **Recommended fix:** Add package options to Stripe as Products with a set price. Track remaining sessions on student records.

### FIN-06 — Revenue model sustainability check
- **Severity:** Low / Advisory
- At current pricing: $90–$125/hr, 6 time slots/day, 5 days/week → maximum gross is ~$2,700–$3,750/week. After court fees, Stripe processing (~2.9% + $0.30), and realistic 60% fill rate, net weekly take-home is approximately $1,400–$2,000. This is viable solo income but leaves little margin for ads, equipment, or growth. **Recommendation:** The $55 Group Clinic should run 4× per week minimum to maximize revenue-per-court-hour. At 4 players × $55 × 4 clinics = $880/week additional from group sessions.

## Section 6: Legal & Compliance

### LGL-01 — No liability waiver (Launch Blocker)
- **Severity:** Critical / Launch Blocker
- **Current state:** There is zero waiver language anywhere in the booking flow. No checkbox, no "I agree to," no linked document.
- **Why this is the #1 legal priority:** Pickleball is a physical sport with real injury risk (Achilles tears, falls, eye injuries from paddle contact). A solo coach operating without a signed waiver has **no contractual defense** if a student sues for an injury that occurs during a session. General liability insurance typically requires a waiver to be primary defense. Without it, DeMario is personally exposed.
- **Recommended fix:**
  1. Draft a liability waiver covering: assumption of physical risk, release of claims for injury, medical clearance self-certification, photo/video release opt-in, and governing law (Texas).
  2. Add a checkbox to the booking modal step 1: "I have read and agree to the [Coaching Agreement & Liability Waiver](#)" — required to advance.
  3. Store a `waiver_signed_at` timestamp and `waiver_version` on each booking record.
  4. Use a service like Docusign, HelloSign, or a simple in-modal HTML waiver with cryptographic timestamp.
  - **Consult a Texas sports/recreation attorney** to draft the waiver language. Cost: $300–$600 one-time. Worth every cent.

### LGL-02 — No Privacy Policy
- **Severity:** Critical
- **Current state:** The site collects name, email, and phone number from every person who attempts a booking. There is no privacy policy page, no cookie disclosure, no data handling description.
- **Why this matters:** CCPA (California Consumer Privacy Act) applies to businesses that collect personal data from California residents — regardless of where the business is located. GDPR applies to EU residents. Both require disclosure of: what data is collected, how it's used, how to request deletion.
- **Recommended fix:**
  - Add `/privacy` page with a policy covering: data collected, use (booking management only), Supabase as processor, retention period, deletion requests (email).
  - Link in footer: "Privacy Policy" next to copyright.
  - Use a free generator like Termly or Iubenda as a starting point; have an attorney review.

### LGL-03 — No Terms of Service
- **Severity:** High
- **Current state:** No TOS page. This means:
  - No enforceable cancellation policy
  - No payment terms (when payment is due, refund eligibility)
  - No dispute resolution clause
  - No limitation of liability clause
- **Recommended fix:** Add `/terms` with: service description, pricing, cancellation policy (24-hour notice / forfeit deposit), refund policy (no refunds on completed sessions), limitation of liability, governing law (Texas), dispute resolution.
- Link in footer and from the booking modal: "By booking, you agree to our [Terms of Service](/terms)."

### LGL-04 — No cookie consent banner
- **Severity:** Medium
- **Current state:** No analytics scripts are installed yet, so no cookies are dropped. But once GA4 or Meta Pixel is added (which it should be), GDPR and CCPA require a consent mechanism.
- **Recommended fix:** Add `next-cookie-consent` or `react-cookie-consent` before adding any analytics. Configure to block analytics scripts until user accepts.

### LGL-05 — No photo/video release for student content
- **Severity:** Medium
- **Current state:** The waiver language (when it exists) should include an optional photo/video release. Currently there is no consent mechanism for posting student footage on Instagram/TikTok.
- **Recommended fix:** Include in the booking waiver: "Optional: I consent to DeMario Montez using photos/videos from my sessions for coaching promotion on social media. ☐ Yes ☐ No."

### LGL-06 — No parental consent for minors
- **Severity:** High (if coaching juniors)
- **Current state:** No age gate, no minor indicator, no parental consent flow in booking.
- **Recommended fix:** Add a "Is this booking for a minor (under 18)?" checkbox. If yes, require a parent/guardian name field and add a minor-specific consent section to the waiver requiring adult signature.

### LGL-07 — Business entity not disclosed
- **Severity:** Low / Advisory
- **Current state:** Footer reads "DeMario Montez · Pickleball Coach" with no LLC or "DBA" designation.
- **Recommended fix:** If operating as an LLC, display "DeMario Montez Coaching LLC" or similar. If sole prop, a DBA registration provides some brand protection. Texas LLC filing is ~$300 and provides personal liability separation — strongly recommended for a physical-activity business.

### LGL-08 — WCAG 2.1 AA accessibility gaps
- **Severity:** Medium (real legal exposure — ADA Title III applies to public websites)
- **Current state:**
  - No `:focus-visible` styles — keyboard navigation has invisible focus ([src/app/globals.css](src/app/globals.css))
  - No `@media (prefers-reduced-motion)` — auto-rotating carousel and scroll animations cannot be disabled
  - Gallery images are CSS backgrounds — no `alt` text for screen readers
  - Testimonial carousel auto-rotates with no pause control (WCAG 2.2.2)
- **Recommended fix:**
  ```css
  /* globals.css — add these */
  :focus-visible { outline: 2px solid var(--accent); outline-offset: 3px; border-radius: 4px; }
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
  }
  ```
  Convert gallery images from CSS backgrounds to `<img>` tags with descriptive `alt` attributes.
